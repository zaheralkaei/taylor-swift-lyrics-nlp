# Song vibes — embedding clusters

Generated by `python analyze/vibes.py`. K-means clustering (K=10) on
the sentence-transformers/all-MiniLM-L6-v2 song embeddings from phase 5. No new model —
embeddings are cached at `data/processed/song_embeddings.npz`.

**Note**: this is a lightweight replacement for an LLM-based vibe
labeling pass (qwen3:4b via ollama, phase 6 in the original plan).
The clusters are real and meaningful; the 'vibe' hints are
heuristic (top songs + dominant album), not LLM-generated prose.
For per-song prose summaries, an LLM pass would still be needed.

## Within-album cluster consistency

For each album, how many of its songs share a cluster with another
song from the SAME album? Higher = album is internally coherent.

| Album | n songs | in shared cluster | consistency % |
|-------|---------|-------------------|---------------|
| TTPD | 31 | 30 | 97% |
| Red | 28 | 27 | 96% |
| Fearless | 25 | 24 | 96% |
| 1989 | 22 | 21 | 95% |
| Speak Now | 22 | 19 | 86% |
| Midnights | 21 | 18 | 86% |
| Folklore | 17 | 14 | 82% |
| Evermore | 17 | 14 | 82% |
| Lover | 18 | 13 | 72% |
| Taylor Swift | 14 | 12 | 86% |
| Reputation | 15 | 11 | 73% |
| Life of a Showgirl | 12 | 8 | 67% |
| Other | 2 | 0 | 0% |

**Caveat on consistency**: the variation 67% to 97% mostly tracks
album size (n=12 to n=31) — a 31-song album is more likely to land
multiple songs in the same cluster than a 12-song album, by chance.
This is a real signal (TTPD songs are more similar to each other) but
it's confounded with sample size.

## Cluster quality (computed dynamically)

K-means (K=10, seed=42) on 244 song embeddings:

| Metric | Value |
|--------|-------|
| Silhouette score (mean) | 0.0038 |
| Silhouette score (median) | 0.0001 |
| Fraction of songs with negative silhouette | 49.6% |
| Fraction of songs with silhouette > 0.3 | 0.0% |
| ARI between K=10 clusterings at different seeds | 0.131 to 0.204 |
| Pairwise cosine similarity (mean ± std) | 0.437 ± 0.102 |

**What this means**: the 384-dim sentence embeddings place all 244 songs
in a relatively tight region of space (all pairwise similarities are
positive, mean ≈ 0.4). K-means splits this region into K pieces, but the
pieces don't correspond to meaningful 'vibe' categories — the cluster
labels are arbitrary. A different random seed would give a different set
of 10 clusters with ARI ≈ 0.20 against the current ones, telling
a different 'story' with the same data.

The cluster compositions (top 5 songs, dominant album) shown above are
real for seed=42 but seed-dependent. The summary's within-album
consistency % is also affected — different seeds give different ranks.
Treat the cluster descriptions as exploratory, not definitive.

## Reproducing

```bash
python data/fetch_cots.py
python data/build_pipeline.py
python analyze/similarity.py    # writes song_similarity.csv (also caches embeddings)
python analyze/vibes.py         # this file
```

Output files:
- `reports/song_vibes.csv` — 244 rows × cluster assignment (committed)
- `reports/vibes_summary.md` — this file (committed)
- `data/processed/song_embeddings.npz` — cached embeddings (gitignored)
