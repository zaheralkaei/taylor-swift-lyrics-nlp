# Song similarity — summary

Generated by `python analyze/similarity.py`. Encodes each song's lyrics with
`sentence-transformers/all-MiniLM-L6-v2` (sentence-transformer, ~80 MB) and finds the top-5 most
similar songs by cosine similarity.

**Notation**: similarity scores are cosine similarity in [0,1]
(embeddings are L2-normalized, so dot product = cosine).
Higher = more semantically alike.

## Cross-album similarity

Of 244 songs, **210** (86%) have their nearest
neighbor from a *different* album. High cross-album similarity
suggests Swift draws on a shared lyrical pool across eras —
a song on Folklore can be closest to a song on 1989, etc.

## Per-album average centroid similarity

For each album, compute the mean of its song embeddings (centroid)
and the cosine similarity to every other album's centroid.
Closest-neighbor album pairs cluster most strongly in lyrical
content (even if their genres differ).

## Reproducing

```bash
python data/fetch_cots.py
python data/build_pipeline.py
python analyze/similarity.py
```

Output files (gitignored, regenerable):
- `reports/song_similarity.csv` — 244 rows × top-5 neighbors + scores
- `reports/similarity_summary.md` — this file

First run downloads the sentence-transformer model (~80 MB) into
`~/.cache/huggingface/` (HF default cache). Subsequent runs reuse it.
