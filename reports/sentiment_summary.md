# Sentiment analysis — summary

Generated by `python analyze/sentiment.py` on 244 songs (CoTS v1.4 corpus).
Full per-song CSV: `reports/sentiment_per_song.csv` (regenerable, gitignored).

## Models used

| Model | Type | Output |
|-------|------|--------|
| VADER | Rule-based lexicon | neg/neu/pos + compound [-1,1] |
| TextBlob | Rule-based lexicon | polarity [-1,1] + subjectivity [0,1] |
| DistilBERT (SST-2 fine-tune) | Neural classifier | positive/negative probability |

Note: an earlier version of this pipeline also ran a 7-way emotion
classifier (j-hartmann/emotion-english-distilroberta-base). It was
removed because the model is trained on short-form reviews/dialogue
and produces near-uniform distributions on abstract lyrical text —
not a reliable signal. DistilBERT transfers well to lyrics.

## Overall sentiment (244 songs)

| Model | Mean | Median | Min | Max |
|-------|------|--------|-----|-----|
| VADER compound | +0.450 | +0.970 | -1.000 | +1.000 |
| TextBlob polarity | +0.097 | +0.102 | -0.625 | +0.575 |
| DistilBERT pos prob | +0.327 | +0.017 | +0.000 | +0.999 |

**Reading**: VADER is systematically optimistic (median +0.97 — it catches
words like "love", "beautiful", "kiss" but ignores surrounding context).
DistilBERT is polarised (median 0.02 — most songs push strongly positive
OR strongly negative, little middle ground). TextBlob is the most
calibrated, with values clustered around zero.

## Reproducing

```bash
python data/fetch_cots.py
python data/build_pipeline.py
python analyze/sentiment.py
```

Output files (gitignored, regenerable):
- `reports/sentiment_per_song.csv` — 244 rows × 14 sentiment columns (VADER + TextBlob + DistilBERT)
- `reports/sentiment_per_section.csv` — 717 rows (one per song × section)
- `reports/sentiment_summary.md` — this file

**Reproducibility note**: `data/fetch_cots.py` downloads CoTS from
upstream's `main` branch HEAD. If the upstream corpus changes,
results will silently differ. For paper-grade reproducibility, pin
to a specific commit SHA + add a SHA256 check.
